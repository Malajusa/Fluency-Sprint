# Lesson-first fluency architecture

Planned update 06.

## Instructional philosophy

The site should treat fluency as rapid, accurate, durable and usable recall.

Speed is useful, but it is not the goal by itself. Students need explicit teaching, worked examples, guided retrieval, spaced review, corrective feedback and diagnostic next steps.

## Core sequence

1. Explicit teaching
2. Worked example
3. Guided retrieval
4. Independent fluency practice
5. Sprint/diagnostic
6. Feedback and next practice recommendation

## Practice pathway

When a student chooses `Practise this skill`, the site should open a Mr Slay mini lesson before fluency practice.

The mini lesson should include:

- Mr Slay as the explicit teacher
- I do / We do / You do / Sprint later routine
- visual model
- worked examples
- guided practice steps
- common mistake warning
- fluency target
- start practice button

## Practice mode

Practice mode should include persistent Mr Slay support.

Students should be able to open a worked example during practice.

## Sprint mode

Sprint mode should remain assessment-like.

No live Mr Slay support during sprint mode.

## First-pass lesson coverage

The first implementation should cover strong templates for:

- add within 10
- friends of 10
- bridge to 10
- large friends of numbers
- subtract from 10
- bridge through 10
- multiplication strategies
- division fact families
- metric conversion/place-value shift
- mixed operation recognition

## Future expansion

Future updates should expand the lesson bank into all fluency areas, including fractions, percentages, area, angle rules, equations, expanding brackets and index laws, while keeping the core site focus on fluency rather than broad textbook instruction.
