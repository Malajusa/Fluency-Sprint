# Times Tables and Related Division Facts: Pedagogical Root

## Purpose

This document is the pedagogical root for Fluency Sprint's approach to times tables and related division facts.

It should guide:

- multiplication fact pathways
- related division fact pathways
- student-facing lesson language
- staged practice modes
- Mr Slay advice
- diagnostics
- progress tracking
- review and maintenance
- future teacher observations and dashboard insights

## Bottom line

Times-table mastery should not begin with memorise harder.

The strongest model is:

```text
meaning
-> visual structure
-> skip-counting
-> derived facts
-> ordered recall
-> random recall
-> mixed recall
-> related division
-> maintenance
```

Rote recall is the end point, not the starting method.

Students need to understand multiplication as equal groups, arrays, repeated jumps and factor relationships before speed becomes useful.

The best model is not conceptual understanding instead of recall, or recall instead of understanding. It is:

```text
understanding first
strategy next
recall after that
automaticity last
```

## What mastery means

A student has mastered a multiplication fact when they can do all of the following.

| Component | Description | Example |
| --- | --- | --- |
| Represent it | Show the fact with a model. | Show `6 x 4` as 6 groups of 4, 4 groups of 6, an array, or jumps on a number line. |
| Explain it | Say what the fact means and how it works. | `6 x 4 is six groups of four. I can also use 5 x 4 and add one more group of 4.` |
| Derive it | Work it out from known facts when needed. | `7 x 8 = 5 x 8 + 2 x 8.` |
| Recall it accurately | Remember it correctly, first in order and then out of order. | Answer `7 x 8` accurately without starting from the beginning of the table. |
| Recall it automatically | Retrieve it fast enough that it does not consume working memory. | Use `7 x 8 = 56` during division, fractions, area, multi-digit multiplication or problem solving. |

Automaticity matters because it frees working memory for more complex mathematics. However, automaticity must be built on meaning, representation and strategy.

## Recommended teaching sequence

| Step | Student goal | Explicit teaching method | Move on when... |
| --- | --- | --- | --- |
| 1. Equal groups | Understand what multiplication means. | Use counters, bundles, arrays, rows, groups and repeated addition. | Student can model `4 groups of 3` and explain what each number means. |
| 2. Skip-counting | Hear and produce the count sequence. | Count forwards and backwards; use number lines and hundreds charts. | Student can skip-count without losing the sequence. |
| 3. Build the table in order | Connect count sequence to facts. | Record `1 x n`, `2 x n`, `3 x n` beside the skip-counting sequence. | Student can say the table in order and point to matching products. |
| 4. Visual structure | See patterns, not isolated answers. | Use arrays, grid patterns, hundred charts and commutativity. | Student can match `3 x 4` and `4 x 3` and explain why they are equal. |
| 5. Derived facts | Use known facts to find unknown facts. | Teach doubling, halving, 5-plus, 10-minus, near facts and commutativity. | Student can solve unknown facts without counting from 0. |
| 6. Ordered recall | Begin memorising. | Chant, cover-say-check, teacher prompt, student response. | Student recalls the table in order with minimal hesitation. |
| 7. Random recall within one table | Break dependence on order. | Flashcards, oral questioning, mini whiteboards, low-stakes retrieval, app practice. | Student answers facts from that table out of order accurately. |
| 8. Mixed recall | Distinguish between tables. | Interleave a small number of known tables, then gradually widen. | Student recalls across tables without reverting to long skip-counting. |
| 9. Related division facts | Link multiplication and division. | Use fact families, arrays with missing factors and inverse reasoning. | Student uses multiplication to solve division facts. |
| 10. Automaticity and maintenance | Keep facts available long-term. | Short retrieval practice, spaced review and timed fluency only after understanding. | Student remains accurate days and weeks later. |

## Start with meaning: equal groups

Teach multiplication first as groups of equal size, not as `times means bigger` or `just repeated addition`.

For example:

```text
3 x 4 means 3 groups of 4.
Here are 3 groups.
Each group has 4.
Altogether, there are 12.
```

Students should physically build it, draw it and connect it to the equation:

```text
4 + 4 + 4 = 12
3 x 4 = 12
```

Use both language forms:

```text
3 groups of 4
4, three times
```

This matters because later confusion often comes from students not knowing what the two factors represent.

## Use arrays early

Arrays are one of the strongest representations for times tables because they show:

- equal groups
- rows and columns
- commutativity
- area foundations
- division structure
- distributive thinking

For `4 x 6`, students should see four rows of six and then rotate or reinterpret the array:

```text
4 x 6 = 24
6 x 4 = 24
```

This helps students see that `4 x 6` and `6 x 4` are not unrelated facts.

## Teach skip-counting, but do not stop there

Skip-counting is a bridge, not mastery.

For the 3s:

```text
3, 6, 9, 12, 15, 18, 21, 24, 27, 30
```

Then connect each count to a fact:

| Count | Fact |
| --- | --- |
| 3 | `1 x 3` |
| 6 | `2 x 3` |
| 9 | `3 x 3` |
| 12 | `4 x 3` |

The common problem is that students can chant `3, 6, 9, 12...` but cannot answer `7 x 3` without starting from 3 and counting all the way up. That is not automatic recall. It is still a counting strategy.

Teaching progression:

```text
Count by 3s.
What is the 6th number in the 3s pattern?
What is 6 x 3?
What is 3 x 6?
What multiplication fact matches 18?
```

## Teach facts by structure

The site should not treat all tables as equally arbitrary. Facts should be taught by structure.

Recommended sequence:

```text
2s, 5s, 10s
-> 3s
-> 4s
-> 8s
-> 6s
-> 9s
-> 7s
```

Rationale:

- 2s connect to doubles.
- 5s connect to counting by fives and half of ten.
- 10s connect to place value.
- 3s build early skip-counting and grouping fluency.
- 4s are double the 2s.
- 8s are double the 4s.
- 6s can be double the 3s or 5 groups plus 1 group.
- 9s can be 10 groups minus 1 group.
- 7s are usually hardest because they have fewer obvious patterns, so they should lean heavily on derived facts and commutativity.

## Table-specific explicit strategies

### 2 times table

Core idea: doubling.

Teach:

```text
2 x 6 means double 6.
6 x 2 means two groups of 6.
```

Student language:

```text
2 x 7 is double 7, so it is 14.
```

### 3 times table

Core idea: skip-counting, arrays and near facts.

Useful strategies:

```text
5 x 3 = 15, so 6 x 3 = 18.
10 x 3 = 30, so 9 x 3 = 27.
```

Student language:

```text
I know 5 threes is 15. One more three is 18.
```

### 4 times table

Core idea: double the 2s.

Teach:

```text
4 x 6 is double 2 x 6.
2 x 6 = 12, so 4 x 6 = 24.
```

Student language:

```text
Four groups is double two groups.
```

### 5 times table

Core idea: five pattern and half of ten.

Teach:

```text
Products end in 5 or 0.
5 x 8 is half of 10 x 8.
10 x 8 = 80, so 5 x 8 = 40.
```

Student language:

```text
Five groups is half of ten groups.
```

### 6 times table

Core idea: 5 groups plus 1 group, or double the 3s.

For `6 x 7`:

```text
5 x 7 = 35
1 x 7 = 7
35 + 7 = 42
```

Or:

```text
3 x 7 = 21
double 21 = 42
```

Student language:

```text
Six groups is five groups and one more group.
```

### 7 times table

Core idea: use known facts, not chanting alone.

For `7 x 8`:

```text
5 x 8 = 40
2 x 8 = 16
40 + 16 = 56
```

For `7 x 9`:

```text
10 x 7 = 70
subtract one group of 7
70 - 7 = 63
```

Student language:

```text
I do not need to count all the way. I can use a fact I already know.
```

### 8 times table

Core idea: double the 4s.

For `8 x 6`:

```text
4 x 6 = 24
double 24 = 48
```

Student language:

```text
Eight groups is double four groups.
```

### 9 times table

Core idea: 10 groups minus 1 group.

For `9 x 6`:

```text
10 x 6 = 60
subtract 6
54
```

Student language:

```text
Nine groups is one less group than ten groups.
```

This generalises better than isolated finger tricks.

## How recall should be practised

### Stage 1: supported recall

Use:

- visible multiplication chart
- arrays
- number lines
- skip-counting strips
- teacher modelling
- think-aloud strategy use

Do not time this stage. Timing too early rewards guessing, anxiety and shallow associations.

### Stage 2: recall in order

Students practise facts in sequence:

```text
1 x 3 = 3
2 x 3 = 6
3 x 3 = 9
```

This builds sequence familiarity. Ordered chanting is not mastery by itself.

### Stage 3: recall out of order within one table

Example:

```text
4 x 3
9 x 3
6 x 3
2 x 3
8 x 3
```

This is where students begin retrieving facts instead of reciting the whole table from the start.

### Stage 4: mixed-table recall

Only mix tables once the individual tables are reasonably secure.

Start with a small mix:

```text
2s, 5s, 10s
```

Then widen:

```text
2s, 3s, 4s, 5s, 10s
```

Then:

```text
include 6s, 8s, 9s
```

Then:

```text
include 7s
```

Interleaving is useful once students have enough initial knowledge to avoid overload.

### Stage 5: automatic recall

Timed work belongs here, not earlier.

Timed fluency should be:

- brief
- low-stakes
- used after conceptual and strategy instruction
- paired with feedback and review
- not used to introduce concepts

A practical benchmark:

| Level | Response expectation |
| --- | --- |
| Developing | Correct with visual or strategy support. |
| Strategy-secure | Correct within about 8-10 seconds using a known strategy. |
| Recall emerging | Correct within about 5-6 seconds. |
| Automatic | Correct within about 2-3 seconds, without counting. |

There is no single universal research-agreed cut-off for automaticity. A 6-second policy benchmark can be useful, but it should not be treated as the final definition of mastery.

## Corrective feedback

When a student gives a wrong answer, do not simply repeat `try again`. That can rehearse the wrong association.

Use this sequence:

1. Give the correct answer.
2. Prove it with a representation or strategy.
3. Say the complete fact.
4. Have the student repeat the complete fact.
5. Return to the fact after two or three other questions.

Example:

```text
The answer is 42.
Let's prove it.
6 x 7 is 5 x 7 plus 1 x 7.
That is 35 + 7.
So 6 x 7 = 42.
Say it: 6 x 7 = 42.
```

## Daily lesson structure

For a 10-minute daily times-table session:

| Time | Focus | Example |
| --- | --- | --- |
| 1 minute | Review known facts. | Quick oral retrieval of already-secure facts. |
| 2 minutes | Teach one structure. | `6s can be made from 5 groups plus 1 group.` |
| 2 minutes | Represent it. | Arrays, counters, number line or grid. |
| 2 minutes | Guided practice. | `6 x 4? 5 x 4 is 20, plus 4 is 24.` |
| 2 minutes | Independent retrieval. | Mini whiteboards, partner cards, or app practice. |
| 1 minute | Exit check. | One easy fact, one target fact, one mixed review fact. |

This is better than one long weekly worksheet because recall needs frequent, spaced, low-stakes retrieval.

## Website implication

The student-facing site should stage times-table skills like this.

| Website stage | Student-facing goal |
| --- | --- |
| 1. Build it | I can make groups and arrays. |
| 2. Count it | I can skip-count the table. |
| 3. See the pattern | I can spot helpful patterns. |
| 4. Use a smart strategy | I can work out tricky facts without counting from the start. |
| 5. Say it in order | I can say the table in order. |
| 6. Answer shuffled facts | I can answer facts from this table in any order. |
| 7. Mix it with other tables | I can answer facts when different tables are mixed. |
| 8. Use division facts | I can use multiplication to solve division facts. |
| 9. Keep it automatic | I can still answer quickly after time has passed. |

## Unlock principle

Do not unlock timed mixed recall until the student has passed the conceptual and strategy stages.

This matters because timed mixed recall is a fluency check, not a teaching method.

## Product design rules

### Student-facing pathway pages

Student-facing pages should contain student-facing language only.

Do not include:

- teacher notes
- assessment rationale
- research commentary
- dashboard observations
- professional terminology unless student-friendly

Teacher-facing material belongs in:

- documentation
- teacher dashboards
- reports
- analytics views
- planning notes

### Mr Slay advice

Mr Slay should match the student's current stage.

For the same fact, advice should change depending on the stage.

Example for `8 x 3`:

| Stage | Advice |
| --- | --- |
| Build it | Make 8 groups of 3. Count the groups. |
| Count it | Count by 3s until the 8th count. |
| Use a smart strategy | Use `4 x 3 = 12`, then double it. |
| Answer shuffled facts | Build from a fact you already know. |
| Quick recall | If this is slow, go back to strategy practice. |
| Use division facts | Connect it to `24 / 3 = 8` and `24 / 8 = 3`. |

### Error feedback

Error feedback should teach, not just mark.

Avoid:

```text
Wrong. Try again.
```

Prefer:

```text
Not yet. The answer is 24.
Let's prove it.
8 groups of 3 is double 4 groups of 3.
4 x 3 = 12, so 8 x 3 = 24.
```

### Maintenance

Fact recall should not be considered finished after one successful sprint.

The site should include spaced review and maintenance checks so facts remain available over time.

## Decision record

Fluency Sprint's approach to times tables and related division facts is:

> Teach meaning first, strategy next, recall after that, and automaticity last. Timed recall is a later fluency and maintenance tool, not the starting method.
